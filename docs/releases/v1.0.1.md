# ClawItUp v1.0.1

`v1.0.1` is a reliability release for the demo-ready audit path.

## Highlights

- Narrows installed agent tools to read-only audit essentials.
- Adds runtime disallow lists for mutation, memory, and generic helper tools.
- Clarifies that the orchestrator is a scoped handoff stage, not a long-running controller.
- Requires Red Team and Filter to return `observedFiles` backed by successful `read` tool results before candidate or verified findings are accepted.
- Updates scaffolded skills and workflow prompts so installed target repos get the same agent contracts.
- Lets `install.sh` pass a preferred model into `clawitup init --force`, preventing stale target repo configs after local upgrades.

## Verification Targets

- `npm test`
- `npm run typecheck`
- `npm run build`
- `npm pack --dry-run`
- Real target repo smoke run with artifact inspection under `runs/<run-id>/`

## Verified This Release

- Disposable install smoke wrote v1.0.1 target config, tools, skills, workflow, and memory files.
- Live mdview run `20260524163019027` completed with `Policy: WARN` and read-backed `observedFiles` in Red Team and Filter artifacts.
- `npm audit` still reports transitive advisories through the current `gitclaw` dependency tree; npm's suggested fix downgrades `gitclaw` to `0.5.0`, so this release does not take that breaking dependency change.
